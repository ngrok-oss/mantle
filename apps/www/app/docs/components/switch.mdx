# Switch

A control that allows the user to toggle between checked and not checked.

Pair with `Field.*` for label/description/error wiring in forms:

<Example>
	<Field.Item name="airplane-mode-field">
		<Field.Label className="inline-flex items-center gap-2">
			<Field.Control>
				<Switch />
			</Field.Control>
			Airplane Mode
		</Field.Label>
		<Field.Description>Disables wireless radios while in flight.</Field.Description>
	</Field.Item>
</Example>

```tsx
import { Field } from "@ngrok/mantle/field";
import { Switch } from "@ngrok/mantle/switch";

<Field.Item name="airplane-mode-field">
	<Field.Label className="inline-flex items-center gap-2">
		<Field.Control>
			<Switch />
		</Field.Control>
		Airplane Mode
	</Field.Label>
	<Field.Description>Disables wireless radios while in flight.</Field.Description>
</Field.Item>;
```

## Multi-line label with description

For a switch with an emphasized title and a supplementary description beneath it, compose it inside [`Choice`](/components/choice): drop the `Switch` into `Choice.Indicator` and pair `Choice.Label` (a real `<label>` — clicking it flips the switch, since a `<button role="switch">` is a labelable element) with an optional `Choice.Description`. The switch aligns to the leading of the title's first line, and the description associates via `aria-describedby`.

<Example>
	<div className="w-full max-w-md">
		<Choice.Root name="email-notifications">
			<Choice.Indicator>
				<Switch defaultChecked />
			</Choice.Indicator>
			<Choice.Content>
				<Choice.Label>Email notifications</Choice.Label>
				<Choice.Description>
					Get notified by email when a deploy finishes or a tunnel goes offline.
				</Choice.Description>
			</Choice.Content>
		</Choice.Root>
	</div>
</Example>

```tsx
import { Choice } from "@ngrok/mantle/choice";
import { Switch } from "@ngrok/mantle/switch";

<Choice.Root name="email-notifications">
	<Choice.Indicator>
		<Switch defaultChecked />
	</Choice.Indicator>
	<Choice.Content>
		<Choice.Label>Email notifications</Choice.Label>
		<Choice.Description>
			Get notified by email when a deploy finishes or a tunnel goes offline.
		</Choice.Description>
	</Choice.Content>
</Choice.Root>;
```

Wrap that same `Choice` in a `Field.Item` / `Field.Control` to bind a form `name` and pick up validation. The field's `id` / `name` / `aria-*` flow onto the switch, and the `Field.Description` (or a `Field.Errors`) is merged into the control's `aria-describedby` alongside the `Choice.Description`:

				</Choice.Root>

## Examples

### Switch in a form with client-side validation

In this example, the `Switch` is used in a form with client-side validation. The form is built using [`@tanstack/react-form`](https://tanstack.com/form/latest/docs) and `zod` for validation. The form accepts and validates the input before submission.

export const formSchema = z.object({
	airplaneMode: z.boolean(),
});

export function FormExample() {
	const defaultValues = {
		airplaneMode: false,
	};
	const form = useForm({
		defaultValues,
		validators: {
			onSubmit: formSchema,
		},
		onSubmit: ({ value }) => {
			window.alert(`Form submitted: ${JSON.stringify(value, null, 2)}`);
		},
	});

    return (
    <form
    	className="space-y-4 w-full"
    	onSubmit={(event) => {
    		event.preventDefault();
    		event.stopPropagation();
    		void form.handleSubmit();
    	}}
    >
    	<form.Field name="airplaneMode">
    		{(field) => (
    			<Field.Item name={field.name}>
    				<Field.Label className="inline-flex items-center gap-2">
    					Airplane Mode
    					<Field.Control>
    						<Switch
    							checked={field.state.value}
    							onBlur={field.handleBlur}
    							onCheckedChange={(value) => field.handleChange(value)}
    						/>
    					</Field.Control>
    				</Field.Label>
    				<Field.Errors messages={toErrorMessages(field.state.meta.errors)} />
    			</Field.Item>
    		)}
    	</form.Field>
    	<Button type="submit" appearance="filled" priority="neutral">
    		Submit
    	</Button>
    </form>
    );

}

<Example className="flex-col w-full">

</Example>

## API Reference

### Switch

The `Switch` is built on top of [Radix Switch](https://www.radix-ui.com/primitives/docs/components/switch). It exposes the same API with the following additional props:

| Prop        | Type      | Default | Description                                                            |
| :---------- | :-------- | :------ | :--------------------------------------------------------------------- |
| `readOnly?` | `boolean` | `false` | Makes the switch immutable, meaning the user can not edit the control. |
