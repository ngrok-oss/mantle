Add `Field`, a compound layout primitive for building a single form field. `Field.Root` stacks the existing mantle `Label`, a control (Input, Select, Checkbox, etc.), and helper / error messaging together with consistent `gap-1.5` so help and error text sit tightly under the input. `Field.LabelRow` is a horizontal flex container (`items-center`, `gap-1`) for pairing a `<Label>` with sibling affordances that can't live inside it, like a help-icon trigger. `Field.Help` / `Field.HelpTrigger` / `Field.HelpContent` are thin wrappers over `Popover` + `IconButton` + a default Phosphor `QuestionIcon` so the help-affordance pattern is a one-liner without manually pulling in those primitives — `Popover` (not `Tooltip`) under the hood so the affordance is reachable on touch. `Field.Optional` renders an inline muted "(Optional)" suffix for placement inside the `<Label>`. `Field.Description` renders muted body-color hint text; `Field.Error` renders a single message in `text-danger-600`; `Field.ErrorList` accepts an array (e.g. `field.state.meta.errors` from TanStack Form) and renders one `Field.Error` per truthy entry. When a `Field.Error` follows another `Field.Error`, OR when a `Field.Description` follows a `Field.Error`, the gap automatically collapses to `-mt-1.5` so consecutive errors and the trailing helper read as a single tight block (overridable with any user-supplied margin utility). Group multiple fields together with `Field.Set` + `Field.Legend` + `Field.Group` for a semantic `<fieldset>` with `gap-4` spacing between rows. Every layout part supports `asChild` for swapping the underlying element. The component intentionally does not invent a new label primitive — pair `Field.Root` with the existing `<Label>`.
