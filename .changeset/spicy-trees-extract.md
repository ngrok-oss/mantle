Add a build-time, type-aware component prop-extraction codegen. A new `pnpm run codegen` step (wired into `prebuild`) walks `src/components/*/index.ts`, resolves each primary exported component with the TypeScript checker via ts-morph, and emits a deterministic, drift-guarded `src/__generated__/component-props.json` describing each component's props (rendered type text, required-ness, defaults with their source — cva `defaultVariants` > destructure > `@default` — `@enum` member meanings, `@see` URLs, discriminated-union branch info, and own-vs-inherited classification). The shared schema types (`ComponentPropsArtifact`, `ComponentPropSchema`, `PropEntry`, and friends) are exported from `@ngrok/mantle/types` for downstream consumers (docs manifest, prop tables, offline agent metadata).

The codegen also extracts compound (POJO-namespace) components, emitting one flat artifact entry per callable sub-component keyed by its dotted member path (e.g. `AlertDialog.Content`, `AlertDialog.Trigger`). Each sub-component carries an inheritance label in the existing `ComponentPropSchema.extends` field — derived from the wrapped Radix/Ariakit primitive (`Radix <Lib>.<Member>` / `Ariakit <Member>`) or overridden verbatim via an `@extends` JSDoc tag — or, for intrinsic members, a `hostElement` (mutually exclusive with `extends`). Passthrough sub-components with no own props still emit so consumers can render their inherited-prop prose. This rollout is scoped to `AlertDialog` (the `AlertDialog.Action`/`Cancel` sub-components gain `@extends Button` tags and the `AlertDialog.Content` `preferredWidth` `@default` JSDoc is corrected to `max-w-lg` to match its destructure default).
